\begin{frame}{Specified oracles: il criterio di valutazione}
  \begin{columns}
    \begin{column}{.54\textwidth}
      \begin{center}
        \small\textit{Come si stabilisce automaticamente se il comportamento osservato è una violazione?}
      \end{center}
      \vspace{0.4em}
      \begin{itemize}
        \item Il problema dell'oracolo (Barr et al.\ 2015): generare input è risolto; \alert{valutare il comportamento osservato} è il collo di bottiglia

        \item APIGuard adotta \alert{specified oracles}: criteri codificati \alert{a priori} nella logica di ogni test, ricavati dalla conoscenza del dominio del controllo

        \item Sorgenti per dominio:
              \begin{itemize}
                \item schema OAS \textrightarrow{} conformità strutturale
                \item standard tecnici (NIST, ASVS) \textrightarrow{} TLS, header
                \item configurazione gateway \textrightarrow{} policy, routing
                \item confronto tra ruoli \textrightarrow{} difetti RBAC, BOLA
              \end{itemize}
      \end{itemize}
    \end{column}

    \begin{column}{.43\textwidth}
      \centering
