# Security Assurance for Modern Big Data APIs

Modern data platforms increasingly expose their functionality through REST APIs distributed across microservice architectures, making API security a critical, often overlooked dimension of Big Data infrastructure. Automated API testing has traditionally focused on functional correctness, while security assurance remains underdeveloped. Existing tools operate mainly at the syntactic level of requests and responses, unable to detect vulnerabilities tied to the target's semantic behavior, implicated in several top OWASP API Security categories. Detecting them requires evaluation criteria grounded in security semantics, still an open challenge. Existing solutions trade portability off against depth, and non-deterministic execution undermines reproducibility and integration.

This work proposes a contract-driven approach to REST API security assurance. Leveraging the target's OpenAPI specification, the framework generates syntactically valid probes and derives evaluation criteria from the semantics of the assessed controls, embedding them into deterministic test oracles. It requires only the specification and a config file at runtime, testing any documented API without source-code access. Fixed oracles and deterministic execution guarantee reproducible, operator-independent results, organized into an eight-domain taxonomy scaling with privileges.

The approach was tested on a realistic setup and successfully identified both correct and incorrect behaviors. It also revealed differences between what the API documentation declared and how the system actually behaved.

Overall, this work contributes a structured, implementation-agnostic framework for REST API security assurance, a systematic approach to the oracle problem that derives evaluation criteria from security semantics, and a reliable, deterministic execution process that makes security assessment more practical, repeatable, and ready for integration into automated pipelines.
